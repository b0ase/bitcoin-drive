Manifest File Types (MFT) & Bitcoin Attestation Protocol System (BAPS)

Manifest File Types (.mft)

📁 What are MFT Files?

MFT (Manifest File Type) files are cryptographically secured metadata containers that store information about files uploaded to the Bitcoin Drive network. Each MFT file contains:

  • • SHA-256 hash of the original file
  • • Bitcoin transaction ID for immutable timestamping
  • • IPFS CID for decentralized storage reference
  • • Encryption metadata (algorithm, key derivation)
  • • File sharding information for distributed storage
  • • Access control lists (ACL) for permissions

⚙️ MFT File Structure

Field Type Size Description
Magic Bytes Hex 4 bytes 0x4D465442 (MFTB)
Version Uint16 2 bytes Protocol version
File Hash SHA-256 32 bytes Original file hash
BTC Tx ID Hex 32 bytes Bitcoin transaction hash
IPFS CID String Variable Content identifier
Metadata JSON Variable Additional metadata
Signature ECDSA 71-73 bytes Digital signature

🔐 Cryptographic Security

Every MFT file is signed with the user's Bitcoin private key, ensuring authenticity and preventing tampering.

⛓️ Blockchain Anchoring

File hashes are anchored to the Bitcoin blockchain using OP_RETURN transactions for permanent timestamping.

🌐 Distributed Storage

Files are sharded and distributed across multiple nodes, with locations tracked in the MFT manifest.

Bitcoin Attestation Protocol System (BAPS)

🔗 BAPS Overview

The Bitcoin Attestation Protocol System (BAPS) is a revolutionary protocol that leverages Bitcoin's immutable ledger to create verifiable proofs of data existence, integrity, and ownership. BAPS enables:

  • • Timestamped proofs of file existence
  • • Chain of custody tracking for digital assets
  • • Multi-signature attestations for shared ownership
  • • Smart contract integration via Lightning Network
  • • Zero-knowledge proofs for privacy-preserving verification

🛡️ BAPS Security Features

Merkle Tree Aggregation

Multiple file attestations are aggregated into Merkle trees, allowing thousands of proofs in a single Bitcoin transaction.

Taproot Integration

Leverages Bitcoin's Taproot upgrade for enhanced privacy and more complex attestation scripts.

Lightning Channels

Instant attestations via Lightning Network with on-chain settlement for permanent records.

Threshold Signatures

Schnorr signature aggregation enables efficient multi-party attestations with reduced on-chain footprint.

Time-locked Proofs

Create attestations that become valid only after specific block heights using CheckLockTimeVerify.

Recursive Validation

Chain multiple attestations together for complex verification workflows and audit trails.

⚠️ Important: BAPS attestations are permanent and immutable once confirmed on the Bitcoin blockchain. Ensure all data is correct before submission. Transaction fees apply for on-chain attestations.
