NFT Container File Format (.nft)

The .nft (Network File Type) is a versatile container format designed specifically for BitcoinSV blockchain storage. Similar to how .mov and .avi can contain different video codecs, the .nft format acts as a universal container that can encapsulate various data types and formats.

Supported Internal Formats:

  • ‚Ä¢ .xml - For structured data and configurations
  • ‚Ä¢ .csv - For spreadsheet and tabular data
  • ‚Ä¢ .json - For structured objects and APIs
  • ‚Ä¢ .pdf - For documents and reports
  • ‚Ä¢ .jpg/.png - For images and graphics
  • ‚Ä¢ .mp4/.webm - For video content
  • ‚Ä¢ .mp3/.wav - For audio files
  • ‚Ä¢ Any binary data - Raw data streams

NFT Container Structure:

{
  "version": "1.0.0",
  "container": "NFT",
  "metadata": {
    "created": "timestamp",
    "bsvTxId": "transaction_hash",
    "blockHeight": 123456,
    "size": 1048576,
    "mimeType": "application/vnd.spreadsheet",
    "encoding": "base64"
  },
  "payload": {
    "format": "csv",
    "data": "encoded_content_here",
    "checksum": "sha256_hash"
  },
  "attestation": {
    "protocol": "BAP/1.0",
    "signature": "ecdsa_signature"
  }
}

BitcoinSV (BSV) Technology Stack

Bitcoin Drive is built on BitcoinSV (BSV), the blockchain that maintains the original Bitcoin protocol with unbounded scaling capabilities. BSV provides:

  • ‚Ä¢ Unlimited Block Size - Store files directly on-chain
  • ‚Ä¢ Sub-cent Transaction Fees - Economical for data storage
  • ‚Ä¢ Instant Transactions - 0-conf for immediate access
  • ‚Ä¢ IPv6 Integration - Direct peer-to-peer connectivity
  • ‚Ä¢ Native Data Protocols - Built-in support for large data

‚ö†Ô∏è Important: Bitcoin Drive uses BSV, not BTC. We do NOT use Lightning Network, IPFS, or Taproot. All data is stored directly on the BSV blockchain using native Bitcoin script and OP_RETURN transactions.

BAP - Bitcoin Asset Protocol

BAP (Bitcoin Asset Protocol) enables building Bitcoin-native assets and applications with payments, tokens, and on-chain logic. Key capabilities:

Core Features:

  • ‚Ä¢ BApps - Build full-featured Bitcoin Apps using spreadsheets or any interface
  • ‚Ä¢ UTXO Version Control - Git-like versioning for immutable change tracking on-chain
  • ‚Ä¢ Granular Encryption - Encrypt at file, row, column, or cell level with selective access
  • ‚Ä¢ Tokenized Ownership - Issue shares in assets and distribute dividends automatically
  • ‚Ä¢ Micro-Transaction Pricing - 1/1,000,000th of a penny per cell with BSV's low fees
  • ‚Ä¢ Decentralized Infrastructure - No central servers, pure peer-to-peer exchange

BAP Use Cases:

  • ‚Ä¢ DeFi Calculators - Live settlement and yield calculations
  • ‚Ä¢ DAO Voting Systems - Governance spreadsheets with automatic execution
  • ‚Ä¢ Supply Chain Trackers - Automatic payments on delivery confirmation
  • ‚Ä¢ Scientific Computing - Micropay rewards for computation
  • ‚Ä¢ NFT Marketplaces - Royalty formulas and automatic distribution
  • ‚Ä¢ Prediction Markets - Instant payouts based on outcomes

BAP Asset Classes:

  • üìä Spreadsheets (Data Assets)
    Familiar UI with on-chain version control, cell-level UTXO mapping, selective encryption, and monetization per cell/row/sheet.
  • üìÑ Documents (Knowledge Assets)

  • üéµ Media (Creative Assets)

  • ü™ô Tokens & Shares (Financial Assets)

BAP System Architecture

Application Layer BApps, Spreadsheet UI, HandCash Auth, Data Encryption
Protocol Layer BAP Format, Access Control, Version Management, NFT Containers
Blockchain Layer Bitcoin SV, UTXO Model, Smart Contracts (sCrypt/Run)

BSV Data Protocols

Supported Protocols:

  • B:// - Binary data protocol for large files
  • C:// - Content-addressed storage
  • D:// - Dynamic state protocol
  • BCAT:// - Concatenated data for files > 100KB
  • AIP:// - Author Identity Protocol
  • MAP:// - Magic Attribute Protocol
